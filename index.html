AI1 to AI2 Converter (Version 0.3)

Convert a Project from App Inventor Classic to AI2

This is a first crude pass at instructions. There will be better wording and some images soon. Suggestions welcome!

  1. In the App Inventor Classic (AI1) development environment, load the project you wish to convert, and confirm that you can view it in both the Designer window and Blocks Editor. Ideally, you should also confirm that the program works as expected by testing it on an Android device or an emulator.
  2. Save the AI1 project you wish to convert as a .zip file as follows:
    1. Use My Projects to list your projects;
    2. Click the checkbox of the project you wish to save
    3. Select More Options>Download Source. This will download an AI1 project source .zip file to your computer.
    Important: do not attempt to open or uncompress the .zip file in any way. Some computers and browsers (especially Safari) may be configured to automatically open and uncompress the .zip file. In this case, you will need to configure preferences to turn off this feature.
  3. Use the Choose File button below to navigate to the .zip file you just downloaded. Selecting the file will automatically convert your AI1 .zip project file to an AI2 .aia project file and and download the .aia file to your computer. If errors are reported in the red-outlined error box, see the Troubleshooting section below.
  4. In the AI2 development environment, use Projects>Import project (.aia) from my computer to upload the .aia project file. You can now run and edit your project in AI2!

    If the .aia project cannot be loaded into AI2, or if the uploaded project does not work as you expect, see the Troubleshooting section below.

Suggestions welcome on improving this wording.

  • If there are red error boxes above, something went wrong in the conversion process, and you should report this error (how?).
  • If there are orange error boxes above, there is a problem with your project that you need to fix before it can be converted.
  • If there are yellow warning boxes above, your converted project might be fine, but you should carefully test it.
  • The converter assumes your project is from the most recent version of AI1. If you have an old project, the converter might be able to convert it, but if you experience any problems with conversion, you should upgrade your AI1 project to the most recent version of AI1 before converting it to AI2.

    You can upgrade your AI1 project to the most recent version of AI1 simply by loading your project into the current AI1 development environment and viewing it both in the Designer window and Blocks Editor. This process will automatically perform upgrades that might modify your components and blocks. Then it is safe to download your A11 project as a .zip file.

    Important: Certain upgrades may require you to change your project. If you see blocks outlined in red in the Blocks Editor, you need to replace those blocks before you try to convert your project to A2.

  • If your project loads without error, but does not behave properly. there might be an error in your original project. Sometimes blocks are accidentally disconnected or deleted, which is a common source of these errors. In general, you should confirm that your project works in AI1 before converting it. If your original project doesn't work in AI1, the converted one won't work in AI2!
  • There are some known issues with the current converter that are listed below. You should review these if your converted program doesn't look or behave like you expect.
  • Whom should they contact if they're still having trouble? Should they post on a forum? Should they email lyn directly?

  • Older AI1 projects may contain Notifier.ShowChooseDialog and Notifier.ShowTextDialog without the final cancelable argument. Modify these blocks in AI1 before conversion.
  • There is a None color block in AI1, but not in AI2. However, the converter is able to translate it to a make color block that takes a 4-element list [255, 255, 255, 0] (the last list component is the alpha, or opacity, component; opacity 0 means transparent).
